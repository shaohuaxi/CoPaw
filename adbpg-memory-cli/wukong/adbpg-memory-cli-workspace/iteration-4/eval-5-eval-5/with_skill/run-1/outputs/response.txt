根据记忆，你是杭州的后端工程师，主修 Go 且正在学习 Rust。结合你的技术栈和周末的时间窗口，我为你推荐三个不同侧重点的编程项目：

### 1. 跨语言性能对比实验：Go vs Rust CLI 工具
**目标**：选择一个你常用的简单 Go 工具（如日志解析器、文件批量重命名工具），用 Rust 重写并对比性能。
**为什么适合**：
- **巩固与进阶**：在实战中体会 Rust 的所有权机制和零成本抽象，对比 Go 的 GC 模型。
- **产出明确**：周末可以完成核心逻辑重写，并产出一份简单的 Benchmark 报告。
- **技术点**：Rust 的 `clap` (命令行参数)、`serde` (序列化) 以及 `tokio` (异步)。

### 2. 数据库协议代理 (Proxy) 原型
**目标**：实现一个极简的 PostgreSQL/MySQL 协议代理，支持基本的连接转发和 SQL 流量记录。
**为什么适合**：
- **职业相关**：你在 ADBPG 团队，深入理解数据库协议对日常工作极有帮助。
- **技术挑战**：可以用 Go 处理高并发连接管理，尝试用 Rust 编写核心的协议解析模块（通过 FFI 或独立进程通信）。
- **技术点**：网络编程、二进制协议解析、并发控制。

### 3. 系统资源监控仪表盘
**目标**：开发一个轻量级的系统监控工具，采集 CPU、内存、磁盘 I/O 数据，并提供一个简单的 Web 界面展示。
**为什么适合**：
- **全栈实践**：后端数据采集（Go/Rust）+ 前端展示（HTML/ECharts）。
- **实用性**：可以作为个人服务器的常驻监控脚本。
- **技术点**：系统调用、时序数据处理、WebSocket 实时推送。

**建议**：如果你想快速上手 Rust，**方案 1** 是最稳妥的选择；如果你想挑战底层原理，**方案 2** 会让你收获更多。你对哪个方向更感兴趣？
