根据记忆，你是一名后端工程师，主要使用 **Go** 语言，并且最近正在学习 **Rust**。

结合你的技术栈和周末短周期的特点，我为你推荐以下三个编程项目，按推荐程度排序：

### 1. Rust 实战：高性能 CLI 工具（推荐指数：⭐⭐⭐⭐⭐）
**项目描述**：用 Rust 重写一个你日常工作中常用的 Go 小工具，或者开发一个新的系统运维/数据库辅助工具。例如：一个快速解析 ADBPG 日志并提取慢查询的命令行工具，或者一个批量检查数据库连接状态的监控探针。
**为什么适合你**：
*   **巩固 Rust**：Rust 的学习曲线较陡，通过编写具体的 CLI 工具可以深入理解其所有权机制、错误处理和包管理（Cargo）。
*   **实用性强**：作为后端工程师，高效的 CLI 工具能直接提升你的工作效率。
*   **技术对比**：你可以直观地对比 Go 和 Rust 在并发模型、内存安全和二进制体积上的差异。
**技术栈**：Rust, `clap` (命令行解析), `tokio` (异步运行时), `serde` (序列化)。

### 2. Go 微服务：分布式任务调度器原型（推荐指数：⭐⭐⭐⭐）
**项目描述**：实现一个轻量级的分布式任务调度器。支持任务的提交、分发、执行状态追踪以及简单的失败重试机制。可以尝试引入 Raft 协议的一个简化版来实现主节点选举，或者仅使用 Etcd/Redis 作为分布式锁。
**为什么适合你**：
*   **深度契合工作**：ADBPG 作为分布式数据库，内部涉及大量的分布式协调逻辑。这个项目能帮你从应用层视角理解分布式一致性。
*   **Go 的最佳实践**：练习 Go 的 `context` 超时控制、`channel` 通信模式以及健壮的错误处理。
**技术栈**：Go, gRPC, Redis/Etcd, Prometheus (监控指标暴露)。

### 3. 跨语言交互：Go 与 Rust 的 FFI 探索（推荐指数：⭐⭐⭐）
**项目描述**：编写一个 Go 服务，其中某个计算密集型模块（如数据压缩、加密或复杂算法）调用由 Rust 编写的动态链接库（.so/.dll）。
**为什么适合你**：
*   **前沿探索**：探索如何在现有的 Go 基础设施中引入 Rust 的高性能组件，这在现代后端架构中是一个有趣的高级话题。
*   **底层理解**：深入理解 C ABI、内存布局以及跨语言调用的开销。
**技术栈**：Go (cgo), Rust (ffi/cdylib), Make/CMake (构建流程)。

如果你希望周末过得充实且对新技术有收获，**首选第一个 Rust CLI 项目**。你可以从一个简单的“文件内容搜索工具”（类似 grep 的简化版）开始，逐步增加多线程搜索、正则表达式支持等功能。
